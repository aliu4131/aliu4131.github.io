Traffic Accidents in Maryland¶

Alex Liu

Introduction¶

Every year, millions of traffic accidents occur in the United States. In 2019, over thirty-three thousand of these were fatal resulting in over thirty-six thousand deaths, with millions more having serious injuries requiring hospitalization. Traffic-related deaths are the leading cause of death among non-senior Americans.

Measures such as traffic control, stricter or more widespread enforcement of traffic laws, and increased patrols can help lower the number of accidents, but are a strain on city budgets. It can therefore be useful to determine if there are times where it is better to deploy more resources and times where less resources can be used, in order to balance costs.

Setup¶

The data that will be used is a public dataset released by the government of Maryland on accidents occuring on facilities owned by the Maryland Transportation Authority or accidents responded to by MDTA police. The date range of the accidents is July 2012 to November 2020, or a period of 101 months (this number will be used later). The dataset can be downloaded as a csv file here: https://opendata.maryland.gov/Public-Safety/MDTA-Accidents/rqid-652u. Place this in the same directory as this notebook.

Then, import the following libraries:

  • Pandas: used for storing and manipulating the data through its dataframe feature
  • Matplotlib: used for creating graphs and charts
  • Scikit-learn: used for regression and machine learning
  • Datetime: used for storing datetimes as objects
  • Statsmodels: used for testing involving statistic models

Cleaning up the data¶

A look through the entire dataset shows inconsistency between dates in the Date and Time columns, and furthermore inconsistent formatting across entries in the Time column. This will need to be standardized to allow for easier analysis later on. The Time column has several different formats, which along with the Date column will be converted into a new DateTime column as follows:

  1. M/D/YYYY H:MM:SS AM/PM - no change
  2. H:MM:SS AM/PM - time will be combined with the Date column
  3. HHMM or HMM - time will be combined with the Date column
  4. YYYY-MM-DDTHH:MM:SS.000 - time will be combined with the Date column
  5. M/D/YYYY - time will be set to midnight

The Accident Type column also has inconsistent formatting with some rows using the full phrase and some rows using abbreviations. Phrases will be converted to abbreviations as follows:

  1. Property Damage Crash, pd - PD
  2. Injury Crash - PI
  3. Fatal Crash - F

There is one row with Accident Type "IS"; as its meaning is not readily apparent it will be ignored.

In [3]:
datetimes = []

for i, row in df.iterrows():
    # standardize datetimes
    date = row['Date']
    time = row['Time'].split()
    try:
        if 'T' in row['Time']: # YYYY-MM-DDTHH:MM:SS.000
            time = row['Time'].split('T')[1]
            time = time.split('.')[0]
            datetimes.append(datetime.strptime(date + ' ' + time, '%m/%d/%Y %H:%M:%S'))
        elif len(time) == 3: # M/D/YYYY H:MM:SS AM/PM
            datetimes.append(datetime.strptime(row['Time'], '%m/%d/%Y %I:%M:%S %p'))
        elif len(time) == 2: # H:MM:SS AM/PM
            datetimes.append(datetime.strptime(date + ' ' + row['Time'], '%m/%d/%Y %I:%M:%S %p'))
        elif len(time) == 1: # HHMM or HMM
            hour = row['Time'][:1] if len(row['Time']) == 3 else row['Time'][:2]
            minute = row['Time'][-2:]
            datetimes.append(datetime.strptime(date + ' ' + hour + ':' + minute, '%m/%d/%Y %H:%M'))
    except: # M/D/YYYY
        datetimes.append(datetime.strptime(row['Time'], '%m/%d/%Y'))
    
    # standardize accident types
    if row['Accident Type'] == 'Property Damage Crash' or row['Accident Type'] == 'pd':
        row['Accident Type'] = 'PD'
    elif row['Accident Type'] == 'Injury Crash':
        row['Accident Type'] = 'PI'
    elif row['Accident Type'] == 'Fatal Crash':
        row['Accident Type'] = 'F'
    elif row['Accident Type'] == 'IS': 
        row['Accident Type'] = None

# insert new Datetime column and remove Date and Time columns
df.insert(len(df.columns), "Datetime", datetimes, True)
df = df.drop(columns=['Date', 'Time'])

Data Representation¶

Creating some simple graphs can be useful for finding potential trends in the data that may be worth looking into, as it is easier when the data are represented visually. One likely trend would be the number of accidents compared to some measure of time.

As there is a relatively low number of accidents on any given day, a graph based on this measure would be largely useless. It is better to instead sum up the number of accidents in each month for each year. Create a simple graph of the number of accidents over time, grouped by month:

Hypothesis Testing¶

Hypothesis testing uses statistics to test whether some claim made about the data is true or not. There are several components to hypothesis testing. The null hypothesis (H0) is the claim to be tested and potentially rejected, while the alternate hypothesis (Ha) is the claim to be accepted if the null hypothesis is rejected. The significance level (α), a number between 0 and 1 and usually small, is used to determine when the null hypothesis should be rejected. This occurs when the p-value returned by the test is less than the significance level.

In this example:

  • H0: August does not have a significantly higher number of crashes (or is not significantly different from mean)
  • Ha: August does have a significantly higher number of crashes (or is significantly different from mean)
  • α: 0.05

As equality is being tested, a two-tailed test is used. A one-tailed test would be used if an inequality was being tested.

In [13]:
# sum the number of accidents in each month for each year
years = {}
for i, row in df.iterrows():
    if row['Datetime'].month == 8:
        year = row['Datetime'].year
        years[year] = years[year]+1 if year in years else 1

mean = len(df) / 101
ztest,pval = stests.ztest(list(years.values()), value=mean, alternative='two-sided')
print(pval)

0.010152453316452831

As the p-value of 0.01 is less than the significant level of 0.05, the null hypothesis can be rejected in favor of the alternative hypothesis, in that August has a significantly higher number of crashes.

Machine Learning¶

Machine learning uses an existing dataset to train a model that can then be used to predict values out of the dataset's range. One of the simpler methods uses linear regression, which will be used for the model here. This example will use variables for both month and year. These are termed interaction variables as they all have some effect on the number of crashes. One will be created for the year and one each will be created for the months.

The dataframe is first grouped by month and year, and preprocessed into the two input arrays for the model:

In [14]:
# group by month and year
monthyears = {}
for i,row in df.iterrows():
    monthyear = str(row['Datetime'].month) + str(row['Datetime'].year)
    monthyears[monthyear] = monthyears[monthyear] + 1 if monthyear in monthyears else 1

# create input arrays
x = []
y = []

# set the interaction variables
for monthyear,count in monthyears.items():
    year = int(monthyear[-4:])
    month = int(monthyear[:1]) if len(monthyear) == 5 else int(monthyear[:2])
    months = [0] * 12
    months[month-1] = year
    x.append([year] + months)
    y.append(count)

# create model
reg = LinearRegression().fit(x,y)
print("Coefficients:")
print(reg.coef_)
print("Intercept")
print(reg.intercept_)

# get residuals for each month and year
residuals = {}
for monthyear,count in monthyears.items():
    year = int(monthyear[-4:])

    predicted = reg.intercept_ + reg.coef_.tolist()[0]*year + reg.coef_.tolist()[month]*year

Conclusion¶

This tutorial has given an introduction to the data science pipeline and an example of how it can be applied in the real world. While analysis of the data was kept fairly brief, the results do seem to indicate that there is a correlation between month and number of accidents, with hypothesis testing conducted on the month of August indicating that the number of crashes during the month is significantly different than the overall average number of crashes. This information can serve as a motivator for the government to conduct additional in-depth analysis to reveal the full extent of this relationship, as it can assist them in better allocation of resources towards lowering the number of traffic accidents.

The use of machine learning can additionally assist the government in more accurate projections of the number of crashes by month and year for the future. While the model used in this example was not too accurate, the consideration of additional factors not found in the dataset such as time of day, weather, and amount of traffic can help improve these analyses and models.
